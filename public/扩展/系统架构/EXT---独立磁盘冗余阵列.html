独立磁盘冗余阵列

独立磁盘冗余阵列

18 min read

RAID 的概念

独立磁盘冗余阵列（Redundant Arrays of Independent Disks）是一种将多个硬盘或 SSD 组合成单个系统的技术，旨在提高性能和/或数据安全性。即使一个硬盘发生故障，数据仍然可以从其他硬盘恢复。

RAID 通过在驱动器之间分散或复制数据，帮助更可靠、更高效地存储数据。不同的 RAID 级别提供不同的速度、存储容量和容错能力组合。

RAID 控制器

RAID 控制器管理多个硬盘，使它们作为一个系统协同工作。它有助于提高速度，并通过处理硬盘故障来增强数据保护。您可以将其视为一个智能管理器，既能提升性能，又能保障数据安全。

RAID控制器有三种类型：

1. 基于硬件的RAID

使用专用物理控制器来管理硬盘。

  • 提供高速和可靠性
  • 可以独立于计算机处理器工作
  • 通常内置于主板或作为单独的卡

可以将其想象为一位船长顺利地管理驾驶。

2. 基于软件的RAID

使用计算机的处理器和内存来管理 RAID。

  • 无需特殊硬件
  • 成本效益高，但可能会降低整体系统性能
  • 比硬件 RAID 慢

充当得像一个得力助手，但要与其他任务分担负担。

3. 基于固件的 RAID（伪 RAID）

内置于计算机的 BIOS/固件，并在启动期间工作。

  • 操作系统加载后需要驱动程序
  • 比硬件 RAID 便宜，但仍然占用 CPU 资源
  • 也称为混合 RAID 或假 RAID

启动助手，系统运行后将工作移交给软件。

为什么要进行数据冗余

数据冗余虽然占用了额外的空间，但却增强了磁盘的可靠性。这意味着，如果磁盘发生故障，如果相同的数据也备份到另一个磁盘上，我们就可以检索数据并继续操作。另一方面，如果数据分散在多个磁盘上，且没有采用 RAID 技术，那么单个磁盘的损坏就可能影响整个数据。

RAID系统的关键评估点

评估 RAID 系统时，应考虑以下关键方面：

1. 可靠性

可靠性指系统容忍磁盘故障，防止数据丢失的能力。

例子：

  • RAID 0 不提供容错功能；如果一个磁盘发生故障，所有数据都会丢失。
  • RAID 5 可以容忍一个磁盘因奇偶校验数据而发生故障。
  • RAID 6 可以处理两个同时发生的磁盘故障。

2. 可用性

可用性指 RAID 系统可运行且可供使用的时间比例。

例子：

  • RAID 1（镜像）即使在单个磁盘发生故障时也允许立即访问数据。
  • RAID 5 和 6 可能会在重建期间降低性能，但数据仍然可用。

3. 性能

性能是衡量 RAID 系统处理数据任务的效率。包括：

  • 响应时间：系统响应数据请求的速度。
  • 吞吐量：系统处理数据的速率（例如，MB/s 或 IOPS）。

关键因素：

  • RAID 级别对性能的影响不同：
  • RAID 0 提供高吞吐量，但没有冗余。
  • RAID 1 通过从镜像磁盘提供数据来提高读取性能，但可能不会显著提高写入性能。
  • RAID 5/6 会引入奇偶校验计算的开销，从而影响写入速度。
  • 工作负载类型（例如顺序读/写操作与随机读/写操作）。

性能权衡：

更高的冗余度通常以较慢的写入速度为代价（由于奇偶校验计算）。

4. 容量

容量是考虑冗余机制后可供用户使用的存储量。

关键计算：对于一组 N 个磁盘，每个磁盘有 B 个块，可用容量取决于 RAID 级别：

  • RAID 0：所有 N×B 块均可使用（无冗余）。
  • RAID 1：可用容量为 B（由于镜像，因此只有一个磁盘的容量）。
  • RAID 5：可用容量为（N-1）×B（用于奇偶校验的一个磁盘的容量）。
  • RAID 6：可用容量为（N-2）×B（两个磁盘的容量用于奇偶校验）。

权衡：

与非冗余设置（RAID 0）相比，更高的冗余度（RAID 5/6）会减少可用容量。

RAID 对底层系统非常透明。这意味着，对于主机系统来说，它就像一个以块的线性阵列形式呈现的单个大磁盘。这使得 RAID 可以取代旧技术，而无需对现有代码进行太多更改。

不同的 RAID 级别

RAID-0（剥离）

RAID-0 通过将数据分割成更小的“块”并分布到多个磁盘上来提高系统性能。此过程称为“条带化”。它通过启用并行读/写操作来提高数据访问速度，但不提供冗余或容错功能。

  • 分布在磁盘上的一组块形成一个“条带”。例如，“0、1、2、3”就是一个条带。
  • 我们不是一次只将一个块放入磁盘，而是可以先处理两个（或更多）块，然后再移动到下一个块。

评估

  • 可靠性： 0

    • 数据没有重复。因此，一旦丢失，区块将无法恢复。

  • 容量： N∗B

    • 整个空间用于存储数据。由于没有重复，因此 N 个磁盘（每个磁盘包含 B 个块）被充分利用。

优点

  • 它很容易实现。
  • 它以更好的方式利用存储容量。

缺点

  • 单个驱动器的丢失可能会导致系统彻底崩溃。
  • 对于关键系统来说，这不是一个好的选择。

RAID-1（镜像）

RAID-1 通过在单独的磁盘上为每个数据块创建相同的副本（镜像）来增强可靠性。这确保即使一个磁盘发生故障，数据仍然可以从其副本访问。虽然这种配置高度可靠，但需要大量的存储开销。

镜像：

  • 每个数据块写入两个（或更多）磁盘。

例如：

  • 块 0 存储在磁盘 0 上，其副本存储在磁盘 1 上。
  • 块 1 存储在磁盘 2 上，其副本存储在磁盘 3 上。

评估（假设 RAID 系统是如图的镜像级别为 2 ）

  • 可靠性： 1 到 N/2

    • 1个磁盘故障肯定可以处理，因为该磁盘的块在其他磁盘上会有重复。如果我们足够幸运，磁盘 0 和 2 发生故障，那么由于这些磁盘的块在磁盘 1 和 3 上都有重复，因此同样可以处理。因此，在最佳情况下，N/2 个磁盘故障是可以处理的。

  • 容量： N∗B/2

    • 只有一半的空间用于存储数据。另一半只是已存储数据的镜像。

优点

  • 它涵盖完全冗余。
  • 它可以提高数据的安全性和速度。

缺点

  • 它非常昂贵。
  • 存储容量较小。

RAID-2（带专用奇偶校验的位级剥离）

RAID-2 是一种特殊的 RAID 级别，它采用位级条带化技术，并结合使用汉明码进行纠错。在这种配置中，数据以位级分布在多个驱动器上，并使用专用的奇偶校验驱动器进行错误检测和纠正。虽然它具有强大的容错能力，但由于其复杂性和成本，在实际应用中很少使用。

优点

  • 在纠错的情况下，它使用汉明码。
  • 它使用一个指定的驱动器来存储奇偶校验。

缺点

  • 由于需要额外的驱动，结构复杂，成本较高。
  • 它需要额外的驱动器来检测错误。

RAID-3（带专用奇偶校验的字节级剥离）

RAID-3 通过在多个驱动器上采用字节级条带化并将奇偶校验信息存储在专用奇偶校验驱动器上来增强容错能力。专用奇偶校验驱动器允许在单个驱动器发生故障时重建丢失的数据。此配置适用于需要高吞吐量顺序数据处理的工作负载，但对于随机 I/O 操作效率较低。

  • 这里磁盘 3 包含了磁盘 0、磁盘 1 和磁盘 2 的奇偶校验位。如果发生数据丢失，我们可以用磁盘 3 来构建。

评估

  • 可靠性：1
    • RAID-3 可以容忍一个磁盘发生故障。丢失的数据可以通过奇偶校验驱动器和剩余的数据驱动器重建。
  • 容量：(N−1)×B
    • 其中 N 为驱动器总数，B 为每个驱动器的块数。其中一个驱动器的容量保留用于存储奇偶校验信息。

优点

  • 可以批量传输数据。
  • 可以并行访问数据。

缺点

  • 它需要额外的驱动器来实现奇偶校验。
  • 对于小尺寸文件，其运行速度较慢。

RAID-4（具有专用奇偶校验的块级剥离）

RAID-4 引入了跨多个磁盘的块级条带化，并结合专用的奇偶校验磁盘来提供容错能力。数据以块为单位写入，并使用 XOR 函数计算出的奇偶校验信息存储在单独的磁盘上。这种设置允许在单个磁盘发生故障时恢复数据，这使得 RAID-4 比 RAID-0 更可靠，但由于依赖专用的奇偶校验磁盘，在写入密集型场景下效率较低。

  • 在图中，我们可以观察到一列（磁盘）专用于奇偶校验。
  • 奇偶校验使用简单的异或 (XOR) 函数计算。如果数据位为 0,0,0,1，则奇偶校验位为 XOR(0,0,0,1) = 1。如果数据位为 0,1,1,0，则奇偶校验位为 XOR(0,1,1,0) = 0。一种简单的方法是，如果 1 为偶数，则奇偶校验为 0；如果 1 为奇数，则奇偶校验为 1。

评估

  • 可靠性：1

    • RAID -4 最多允许恢复 1 个磁盘故障（由于奇偶校验的工作方式）。如果多个磁盘发生故障，则无法恢复数据。

  • 容量： (N−1)∗B

    • 系统中预留一个磁盘用于存储奇偶校验信息。因此，共有 (N-1) 个磁盘可用于存储数据，每个磁盘有 B 个块。

优点

  • 如果最多丢失一个数据，它有助于重建数据。

缺点

  • 当丢失多个数据时，它无法帮助重建数据。

RAID-5（带分布式奇偶校验的块级条带化）

RAID-5 在 RAID-4 的基础上，将奇偶校验信息分布在所有磁盘上，而不是存储在专用的奇偶校验驱动器上。这种分布式奇偶校验显著提高了写入性能，尤其是对于随机写入操作，同时保持了对单个磁盘故障的容错能力。RAID-5 因其在可靠性、性能和存储效率之间的平衡而成为最常用的 RAID 配置之一。

RAID-5工作原理

  • 块级条带化：数据被分成块并以条带形式分布在阵列中的所有驱动器上。
  • 分布式奇偶校验：使用 XOR 函数计算的奇偶校验位以旋转模式分布在所有驱动器上。

例如：

  • 磁盘 0：数据（D1、D2）、奇偶校验（P1）
  • 磁盘 1：数据（D3、D4）、奇偶校验（P2）
  • 磁盘 2：数据（D5、D6）、奇偶校验（P3）

这种轮换确保没有任何一个磁盘承担所有奇偶校验操作的负担，从而减少了瓶颈。

数据恢复：如果单个磁盘发生故障，可以通过对剩余数据块和奇偶校验信息进行异或运算来重建丢失的数据。

评估

  • 可靠性：1

    • RAID-5最多允许恢复 1 个磁盘故障（由于奇偶校验的工作方式）。如果多个磁盘发生故障，则无法恢复数据。这与 RAID-4 相同。

  • 容量： (N−1)∗B

    • 总体而言，存储奇偶校验位所需的空间相当于一块磁盘。因此，共有 (N-1) 块磁盘可用于存储数据，每块磁盘有 B 个块。

优点

  • 可以使用奇偶校验位重建数据。
  • 它使性能变得更好。

缺点

  • 其技术复杂，并且需要额外的空间。
  • 如果两张光盘均损坏，数据将永远丢失。

RAID-6（具有两个奇偶校验位的块级剥离）

RAID-6 是 RAID-5 的高级版本，通过引入双分布式奇偶校验来增强容错能力。这使得 RAID-6 能够同时从最多两个磁盘的故障中恢复，从而对于具有较大阵列的关键系统而言更加可靠。但是，增加的奇偶校验计算可能会影响写入性能。

RAID-6工作原理

  • 块级条带化：数据被分成块并以条带形式分布在阵列中的所有磁盘上。
  • 双重分布式奇偶校验：为每个块计算两组奇偶校验信息，并以旋转模式分布在阵列中的所有磁盘上。

例子：

  • 磁盘 0：数据（D1、D2）、奇偶校验（P1）
  • 磁盘 1：数据（D3、D4）、奇偶校验（P2）
  • 磁盘 2：数据（D5、P1）、奇偶校验（P3）
  • 磁盘 3：奇偶校验 (P2, P3)、数据 (D6)

数据恢复：如果一个或两个磁盘发生故障，则可以使用剩余数据和奇偶校验信息重建丢失的数据。

评估

  • 可靠性：2

    • RAID-6可以容忍两个磁盘同时发生故障，提供比RAID-5更高的容错能力。

  • 容量：(N−2)×B

    • 其中N是磁盘总数，B是每个磁盘的块数。

优点

  • 非常高的数据可访问性。
  • 快速读取数据交易。

缺点

  • 由于双重奇偶校验，其写入数据事务速度较慢。
  • 需要额外的空间。
