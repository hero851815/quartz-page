EXT CPU调度算法

EXT - CPU调度算法

23 min read

调度的基本概念

在多道程序系统中，进程的数量往往多于处理机的个数，因此进程争用处理机的情况在所难免。处理机调度是对处理机进行分配，即从就绪队列中按照一定的算法（公平、高效的原则）选用一行进程并将处理机分配给它运行，以实现进程并发地执行。

调度的层次

高级调度（作业调度）

  1. 它负责从外部存储器（如硬盘、网络等）中选择一个或多个作业（job），并将它们调入内存中，为它们创建进程，并在系统资源充足时将它们放入就绪队列中等待CPU执行。
  2. 简单点来说，高级调度就是内存与辅存之间的调度，对于每个作业，只调入一次，调出一次。

中级调度（内存调度）

  1. 它负责在内存中运行的进程数量和大小之间进行平衡，并控制进程在内存中的位置和状态；
  2. 中级调度通常由操作系统自动执行，其主要任务是将某些处于阻塞状态的进程移动到外部设备或磁盘上，以释放出宝贵的内存资源。
调度比较
高级调度针对整个作业或进程的调度
中级调度针对进程在内存中的状态进行调度。中级调度主要关注内存的使用情况，以保证系统的稳定性和性能

低级调度（进程调度）

  1. 它控制CPU从就绪队列中选择下一个要执行的进程，并将CPU分配给该进程；
  2. 低级调度的目的是尽可能快地完成当前进程的运行，以达到最大的系统吞吐量和响应速度。

进程调度的方式

非抢占式调度方式

它指定了一个进程在运行中不会被操作系统强制中断，只有当这个进程自己完成或者发生某些事件时才会放弃CPU控制权。

应用场景：

  • 通常适用于一些轻量级应用程序，如嵌入式系统、移动设备或者实时数据处理等场景；
  • 因为这些应用程序通常需要快速响应用户输入或者外部事件，而非抢占式调度可以保证当前进程能够完整地执行，避免出现任务切换带来的开销和延迟，提高了响应速度和稳定性。

抢占式调度方式

在这种调度策略下，操作系统可以强制中断正在执行的进程，并将CPU分配给优先级更高的进程。

应用场景：

  • 通常适用于对响应速度要求比较高、需要快速处理事件的应用程序，如实时系统或者多任务操作系统等场景。它可以保证高优先级进程能够及时获得CPU使用权，提高系统的性能和稳定性。

注意

  • 抢占式调度和非抢占式调度并不是对立的，而是可以结合起来使用；
  • 例如，在Windows操作系统中，就采用了两种调度策略：时间片轮转（抢占式）和优先级调度（非抢占式），以提供更好的操作系统服务。

典型的调度算法

先来先服务调度算法（FCFS）

先来先服务（First Come, First Serve）是最简单的 CPU 调度算法之一。顾名思义，它就是按照进程到达就绪队列的顺序进行处理，就像顾客在杂货店排队一样。

FCFS 调度是一种非抢占式算法，这意味着一旦进程开始运行，它就无法停止，直到它主动放弃 CPU（通常是在它终止或执行 I/O 操作时）。此方法按照进程到达的顺序进行调度，而不考虑优先级或其他因素。

工作机制：

  1. 到达：进程进入系统并按照到达的顺序排队。
  2. 执行：CPU 从队列前面取出第一个进程，执行直到完成，然后将其从队列中移除。
  3. 重复：CPU取出队列中的下一个进程，重复执行过程。

FCFS的优势：

  • CPU调度算法的最简单、最基本的形式
  • 每个进程都有机会按照其到达的顺序执行。这确保了任何进程都不会被任意地赋予优先权。
  • 易于实现，不需要复杂的数据结构。
  • 由于进程按照到达的顺序执行，因此不存在饥饿的风险
  • 它非常适合批处理系统，因为在批处理系统中，每个过程的较长时间通常是可以接受的。

FCFS的缺点：

  • 由于 FCFS 是一种非抢占式 CPU 调度算法，它可能会导致较长的等待时间，尤其是在一个长进程先于一个短进程到达的情况下。这被称为“护送效应”，即短进程被迫在长进程之后等待，从而导致执行效率低下。
  • FCFS 中的平均等待时间比其他的要高得多
  • 由于 FCFS 按照任务到达的顺序进行处理，因此如果短任务在长任务之后到达，则可能需要等待很长时间，这会导致混合有长任务和短任务的系统性能不佳。
  • 位于队列末尾的进程必须等待更长时间才能完成。
  • 它不适用于分时操作系统，因为在分时操作系统中每个进程都应该获得相同数量的 CPU 时间。

最短作业优先调度算法（SJF）

最短作业优先（Shortest Job First）是一种调度方法，它选择执行时间最短的等待进程作为下一个执行对象。这种调度方法可以是抢占式的，也可以不是。它显著减少了其他等待执行进程的平均等待时间。

调度的实现：

  • 按照到达时间对所有流程进行排序。
  • 然后选择具有最短到达时间和最短突发时间的进程。
  • 该过程完成后，创建一个进程池（就绪队列），该进程在前一个过程完成之前到达，并从该队列中选择具有最短突发时间的进程。

SJF的优势：

  • SJF 比先来先服务(FCFS) 算法更好 ，因为它减少了平均等待时间。
  • SJF一般用于长期调度。
  • 它适用于批量运行的作业，其中运行时间是已知的。
  • 就平均周转时间 (TAT) 而言，SJF 可能是最佳的。

SJF的缺点：

  • SJF 可能会导致非常长的周转时间或饥饿。
  • 在 SJF 中，必须提前知道作业完成时间。
  • 很多时候，预测即将到来的 CPU 请求的长度变得很复杂。

优先级调度算法

优先级调度算法（Priority scheduling）是操作系统根据进程优先级进行调度的最常见算法之一。每个进程都会根据内存需求、时间需求、其他资源需求或平均 I/O 与平均 CPU 突发时间之比等标准分配一个优先级值。

优先级最高的进程会被优先选择执行。如果有多个进程具有相同的优先级，则按照它们到达的顺序进行调度，遵循“先到先服务”的原则。之后，被选中的进程会被执行，直到完成或被抢占（取决于调度是抢占式还是非抢占式）。

优先级调度可以通过两种方式实现：

  • 非抢占式优先级调度：CPU 不会被从正在运行的进程中夺走。即使更高优先级的进程到来，当前正在运行的进程也会先完成。
  • 抢占式优先级调度：如果有更高优先级的新进程到达，则可以从当前正在运行的进程中夺走 CPU。

高响应比优先调度算法（HRRN）

高响应比优先调度（Highest Response Ratio Next）算法是一种非抢占式算法，其中 HRRN 调度基于一个称为响应率的额外参数进行。给定 N 个进程及其到达时间和突发时间，任务是使用 HRRN 调度算法找出平均等待时间和平均周转时间。

顾名思义，我们需要找到所有可用进程的响应率，并选择响应率最高的进程。它旨在通过平衡进程的等待时间和突发时间，改进先来先服务(FCFS) 和最短作业优先(SJN) 等更简单的算法。一旦选定进程，它将运行直至完成。以下是计算响应率的公式。

响应比率=(W+S)/S W：迄今为止过程的等待时间 S：过程的突发时间

在 HRRN 中，响应率最高的进程将被选中执行。这种方法优先考虑等待时间较长的进程，同时也考虑了突发时间，确保在长时间等待的进程和快速执行的进程之间取得公平的平衡。

HRRN的特点：

  • 最高响应率下一步是一种非抢占式 CPU 调度算法，它被认为是最优调度算法之一。
  • HRRN 的标准是响应率，模式是非抢占式。
  • HRRN 被认为是最短作业优先算法的修改版，可以减少饥饿问题。
  • 与SJF相比，在HRRN调度算法中，CPU被分配给响应率最高的下一个进程，而不是分配给突发时间较短的进程。

HRRN的优势：

  • 公平性：HRRN通过同时考虑进程的等待时间和突发时间来实现公平性。它确保等待时间较长的进程获得优先权，从而有助于防止出现资源匮乏的情况。
  • 更好的响应时间：由于 HRRN 优先考虑等待时间较长的进程，与 FCFS 等更简单的算法相比，它可以为这些进程带来更好的响应时间。
  • 无饥饿：与其他一些调度算法（例如最短作业优先 (SJF)）不同，HRRN 降低了饥饿的风险。即使需要更多 CPU 时间的进程最终也会得到执行。
  • 易于实现：该算法易于理解和实现，因为它只需要计算响应率并选择具有最高比率的过程。
  • 动态优先级：HRRN 算法动态调整优先级，这意味着等待时间较长的进程（但不一定是等待时间最短的作业）最终将获得更高的优先级。

HRRN的缺点：

  • HRRN 在实践中不可行，因为我们无法提前预测每个进程的突发时间。
  • HRRN 需要计算每个进程的响应率，这会增加一些计算开销，尤其是在具有大量进程的系统中。

时间片轮转调度算法

时间片轮转调度（Round Robin Scheduling）是一种按照时间片大小进行调度的算法。每个进程被分配一个固定大小的时间片，即每次只能执行一段固定长度的时间，如果在这个时间内没有完成，则该进程会被挂起，并将CPU控制权转移到下一个进程上。

这种调度方法的主要目标是确保所有进程都有平等的执行机会，促进任务之间的公平性：

  • 进程到达：进程进入系统并被放入队列。
  • 时间分配：每个进程都被赋予一定量的 CPU 时间，称为量子。
  • 执行：进程在分配的时间内使用 CPU。
  • 轮换：如果进程在规定时间内完成，则离开系统。如果没有完成，则回到队列末尾。
  • 重复： CPU 继续循环队列，直到所有进程完成。

调度的优势：

  • 公平性：每个进程都获得平等的 CPU 份额。
  • 简单性：算法简单，易于实现。
  • 响应能力：循环算法可以处理多个进程而不会出现明显的延迟，使其成为分时系统的理想选择。

调度的缺点：

  • 开销：进程之间的切换可能会导致高开销，特别是当量子太小时。
  • 利用率不足：如果量子太大，则会导致 CPU 在等待进程完成时感觉无响应。

多级队列调度（MLQ）

多级队列调度（Multilevel Queue Scheduling）是一种 CPU 调度机制，其中进程被划分为多个层级队列，每个队列拥有不同的优先级和进程类型。每个队列的调度算法可能不同，并且这些进程会根据某些标准（例如优先级或资源）以永久方式映射到特定队列。

MLQ的特点：

  • 多队列：在 MLQ 调度中，进程根据其优先级被划分到多个队列中，每个队列具有不同的优先级。高优先级进程被放置在优先级较高的队列中，而低优先级进程被放置在优先级较低的队列中。
  • 分配优先级：根据进程的类型、特性和重要性为其分配优先级。例如，用户输入/输出等交互式进程的优先级可能高于文件备份等批处理进程。
  • 抢占： MLQ 调度允许抢占，即高优先级进程可以抢占低优先级进程，并将 CPU 分配给高优先级进程。这有助于确保高优先级进程及时执行。
  • 调度算法：每个队列可以使用不同的调度算法，具体取决于该队列中进程的需求。例如，对于交互式进程，可以使用循环调度；对于批处理进程，可以使用先来先服务调度。
  • 反馈机制：可以实现反馈机制，根据进程随时间的变化调整其优先级。例如，如果一个交互式进程在低优先级队列中等待了很长时间，则可以提高其优先级，以确保其及时执行。
  • 高效分配 CPU 时间： MLQ 调度确保及时执行具有较高优先级的进程，同时仍允许在 CPU 空闲时执行较低优先级的进程。
  • 公平性： MLQ 调度根据优先级和要求，为不同类型的进程公平分配 CPU 时间。
  • 可定制： MLQ调度可以定制，以满足不同类型流程的特定要求。

MLQ的优势：

  • 调度开销低：由于进程被永久地分配到各自的队列中，因此调度的开销很低，因为调度程序只需要选择适当的队列进行执行。
  • 高效分配 CPU 时间：调度算法确保高优先级进程及时执行，同时允许低优先级进程在 CPU 空闲时执行。这确保了 CPU 时间的最佳利用。
  • 公平性：调度算法根据不同类型进程的优先级和要求，公平地分配 CPU 时间。
  • 可定制：调度算法可定制，以满足不同类型进程的特定需求。每个队列可以根据其进程的需求使用不同的调度算法。
  • 优先级：根据流程的类型、特点和重要性分配优先级，以确保重要流程及时执行。
  • 抢占：多级队列调度允许抢占，即高优先级进程可以抢占低优先级进程，将CPU分配给高优先级进程，从而保证高优先级进程能够及时执行。

MLQ的缺点：

  • 如果某些高优先级队列始终为空，则某些进程可能会缺少 CPU。
  • 其本质是不灵活的。
  • 实现和维护多个队列和调度算法可能会增加复杂性。

多级反馈队列调度（MLFQ）

多级反馈队列调度（Multilevel Feedback Queue Scheduling）调度类似于多级队列 (MLQ) 调度，但在此过程中可以在队列之间移动。因此，它比多级队列调度效率更高。

MLFQ的特点：

  • 多队列：与 MLQ 调度类似，MLFQ 调度根据进程的优先级将其划分到多个队列中。然而，与 MLQ 调度不同的是，进程可以根据其行为和需求在队列之间移动。
  • 动态调整优先级：进程的优先级可以根据其行为（例如已使用的 CPU 时间或被阻塞的频率）动态调整。高优先级进程将获得更多 CPU 时间，而低优先级进程将获得较少 CPU 时间。
  • 时间分片：每个队列都分配有一个时间段或时间片，这决定了该队列中的进程在被抢占并移至较低优先级队列之前可以使用多少 CPU 时间。
  • 反馈机制： MLFQ 调度使用反馈机制，根据进程随时间的变化调整其优先级。例如，如果低优先级队列中的进程用完了其时间片，则可能会将其移至高优先级队列，以确保获得更多 CPU 时间。
  • 抢占： MLFQ 调度允许抢占，这意味着高优先级进程可以抢占低优先级进程，以确保其获得所需的 CPU 时间。

MLFQ的优势：

  • 它更加灵活。
  • 它允许不同的进程在不同的队列之间移动。
  • 它通过将等待低优先级队列时间过长的进程移动到高优先级队列来防止饥饿。

MLFQ的缺点：

  • 最佳调度程序的选择，需要一些其他方法来选择值。
  • 它会产生更多的 CPU 开销。
  • 它是最复杂的算法。
