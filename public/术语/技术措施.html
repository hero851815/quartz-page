数据接口安全

防护措施

1. 身份鉴别 目标：验证调用方的身份合法性。 实现：

  • API密钥：服务器生成一个唯一的API Key，分配给用户/应用；客户端每次调用API时，在请求头或参数中提供API Key。
  • OAuth 2.0：通过授权服务器颁发访问令牌Access Token。
  • JWT：基于令牌的身份验证，支持携带用户角色和权限。
  • 多因素认证与证书

2. 访问控制 目标：限制接口的访问范围。 实现：

  • IP白名单：仅允许特定 IP 或网段访问敏感接口。
  • RBAC（基于角色的访问控制）：根据用户角色限制接口权限。
  • ABAC（基于属性的访问控制）：通过用户属性、资源属性和环境因素（如时间、地点、设备）进行动态访问控制。

3. 授权策略 目标：动态授予最小必要权限。 实现：

  • OAuth2 Scope 机制：通过Scope控制令牌的权限，如read:profile、write:posts。
  • ACL（访问控制列表）：指定哪些用户/应用能访问特定资源。

4. 接口签名 目标：防止请求被篡改或重放攻击。 实现：

  • HMAC-SH256（基于哈希的消息认证码）：服务器和客户端共享一个密钥，用于生成请求签名。
  • RSA（非对称签名）：客户端使用私钥对请求签名，服务器用公钥验证。
  • 时间戳防重放：请求中携带时间戳，服务端校验时间窗口（如 ±5 分钟）。

5. 安全传输协议 目标：防止数据在传输过程中被窃听或篡改。 实现：

  • HTTPS/TLS 加密：强制所有API请求使用HTTPS，防止中间人攻击。
  • mTLS（双向TLS）：客户端和服务器都使用证书进行身份验证。

网络安全

网络类型

专线网络 定义：专线网络是指企业或机构向运营商租用的专用通信线路，用于实现高质量的网络连接。它可以连接企业的总部、分支机构、数据中心等，提供稳定、高速、安全的网络服务。 特点：

  • 独占带宽：企业拥有独立的物理或逻辑专线，不受公网流量影响。
  • 高安全性：数据不经过公共互联网，降低了被攻击或窃听的风险。
  • 低延迟、高稳定性：适用于高并发、低延迟的应用场景（如金融交易、远程医疗、视频会议）。
  • 高成本：需要租用运营商提供的专线，费用比 VPN 高。 强调网络架构的专用性和物理隔离。

VPN 定义：是一种在公共网络（如互联网）上创建安全加密隧道，实现远程访问或分支机构互联的技术。它允许用户在不租用专线的情况下，实现类似专线的安全通信。 特点：

  • 成本较低：无需租用物理专线，仅需 VPN 服务器和客户端即可使用。
  • 安全性较高：通过加密隧道技术保护数据，防止窃听和篡改。
  • 灵活性强：支持远程办公、跨国访问、分支机构互联。
  • 依赖公网质量：数据传输受互联网波动影响，稳定性较专线差。 强调在公共网络上构建虚拟专用网络，提供安全通信。

网络安全设备

网闸（安全隔离网关） 定义：是一种物理或逻辑设备，用于在两个网络之间实现单向数据传输，确保数据只能从一个方向流动，而不能反向传输。它通常用于高安全需求的环境中，如政府、军事和关键基础设施。 特点：

  • 物理隔离或逻辑隔离，高安全性
  • 单向或双向数据传输控制，确保信息可控流通
  • 协议转换，如文件、数据库、邮件等数据安全交换
  • 无法直接访问隔离网络，防止病毒、黑客入侵 适用场景：
  • 高安全等级网络隔离（物理隔离 / 逻辑隔离）
  • 数据单向传输 / 受控双向传输
  • 防止外部攻击和数据泄露

防火墙 定义：是一种网络安全系统，通过监控和控制进出网络流量，根据预定的安全规则来阻止未经授权的访问。它充当网络边界的第一道防线，保护内部网络免受外部威胁。 特点：

  • 基于规则的访问控制，如 IP、端口、协议过滤
  • 支持状态检测（SPI），可以识别并管理网络连接状态
  • 可集成入侵防御系统（IPS/IDS），检测恶意流量
  • 分为软件防火墙和硬件防火墙。 适用场景：
  • 企业网络边界防护
  • 访问控制，阻止未经授权的流量
  • 防止网络攻击（如 DDoS、SQL 注入、木马入侵）

堡垒机 定义：是一种专门设计用于提供集中化访问控制和审计功能的服务器或设备。它通常位于网络的最前沿，作为外部访问的第一道防线，确保只有经过授权的用户和流量能够进入内部网络。 特点：

  • 集中身份认证（多因素认证），防止账户滥用。
  • 访问控制，精细化管理不同用户对服务器、网络设备的权限
  • 操作审计，完整记录所有运维操作（屏幕录制、命令审计）
  • 支持 SSH、RDP、Telnet、数据库管理等远程运维协议 适用场景：
  • 运维安全管理（身份认证 + 权限管理 + 操作审计）
  • 防止内部人员滥用权限或误操作
  • 满足合规性要求（如金融、医疗、政府法规）

数据采集活动

数据采集（chatgpt）

定义 指通过多种手段和途径从不同的数据源获取所需数据的过程。这一过程的核心目标是确保获得与特定业务需求相关的、合法合规的数据。

常见场景

  1. 从原始数据源获取数据：例如，通过传感器、设备、日志、用户输入等渠道采集未经处理的原始数据。此类数据通常需要进一步处理、清洗和转换才能用于分析或其他业务需求。
  2. 获取结构化和非结构化数据：数据采集不仅限于传统的结构化数据（如数据库中的表格数据），还包括非结构化数据（如文本、图片、视频等）和半结构化数据（如JSON、XML等）。
  3. 从外部数据源获取数据：通过API接口、网络爬虫或与第三方合作获取的数据，通常是公开的或授权的数据。采集此类数据时需要特别注意合规性要求，如获得适当的授权、确保数据隐私等。
  4. 实时数据的采集：例如，通过物联网（IoT）设备或实时监控系统获取的数据，这类数据通常用于及时的分析和决策。
  5. 经过初步处理的数据采集：尽管数据采集通常是获取原始数据，但在某些情况下，也可能采集已经进行初步处理或清洗的数据，这些数据可能来自其他系统或数据仓库。

数据提供活动

数据转移（chatgpt）

定义 指数据从一个位置、系统或网络节点移动到另一个位置、系统或节点的过程。

风险点

  • 传输过程中数据的保密性：未加密数据可能被截取。
  • 完整性风险：数据可能在传输中被篡改，导致数据质量下降或产生错误信息。
  • 可用性风险：数据在转移中丢失、损坏或未能成功恢复，影响业务连续性。

常见场景

  1. 内部数据共享与转移。
    • 在企业或组织内部，不同业务部门之间共享数据。例如，销售部门将客户信息共享给财务部门。
  2. 外部数据传输
    • 将数据提供给外部服务商或合作伙伴，例如将客户信息传输到外部支付平台或分析工具。
  3. 跨境数据转移
    • 数据在不同国家的系统和数据中心之间传输，常见于跨国公司的运营中。

防护措施

  • 加密数据在传输途中的内容。
  • 确保传输端点的身份验证和访问控制。
  • 选择可靠的传输协议（如 HTTPS、SFTP、VPN 等）。
  • 实施数据转移过程中的详细审计和日志记录，以便事后追踪和分析潜在问题。

数据存储活动

逾期存储（chatgpt）

定义 指企业或组织在数据的生命周期管理过程中，未按照既定的合规要求、业务需求或数据管理策略，超出法定或内部规定的存储期限而继续保留数据的行为。逾期存储的数据可能包括个人信息、业务数据、日志、财务记录等，长期保留这些数据会带来合规性、安全性和运营成本方面的风险。

风险

  1. 合规风险
    • 违反法律法规（如《个人信息保护法》（PIPL）、《通用数据保护条例》（GDPR）、《数据安全法》等），可能导致行政处罚、法律责任或企业声誉受损。
    • 监管部门通常要求特定类型的数据（如财务、医疗、个人隐私数据）必须在规定的时间后销毁或匿名化。
  2. 数据泄露风险
    • 过期数据可能包含敏感信息，随着存储时间的延长，暴露给未经授权访问的风险增大。
    • 旧数据存储系统可能缺乏有效的安全措施，易受黑客攻击。
  3. 存储成本增加
    • 不必要的长期存储占用大量存储资源，增加存储和管理成本，影响企业的IT预算。
    • 难以高效管理，影响数据存取效率。
  4. 数据质量下降
    • 过时数据可能导致业务决策失误，影响数据分析和洞察的准确性。
    • 数据冗余会降低数据治理效率，影响数据一致性。

常见场景 5. 日志数据未按规定清理 + 服务器、应用、系统日志长时间存储，未定期归档或删除。 6. 备份数据长期保留 + 业务数据备份未按照数据生命周期策略执行定期删除，导致冗余数据积累。 7. 客户信息未及时销毁 + 过期的用户注册信息、交易记录等未按照隐私政策销毁，增加泄露风险。 8. 项目历史数据长期存储 + 项目完成后仍然保留大量历史文档和数据，未归档或清理。

防护措施

  1. 制定数据生命周期管理策略
    • 明确数据从创建、存储、归档到销毁的周期，设定明确的存储期限。
  2. 实施数据分类分级管理
    • 针对不同类型的数据（如敏感数据、普通业务数据），采取不同的存储策略和保留期限。
  3. 自动化数据清理机制
    • 采用数据存储策略，如定期自动删除、归档、匿名化，减少人工干预的可能性。
  4. 合规审计与检查
    • 定期进行数据合规性审计，确保数据保留策略符合法律法规和行业标准。
  5. 存储优化技术
    • 采用存储压缩、冷热数据分层管理等方式，减少存储占用，同时确保访问效率。

数据传输协议

网络层协议

IPsec（Internet Protocol Security IP安全协议，RFC 4301）

  • 功能：是网络层协议，常与L2TP等链路层协议结合，提供加密和认证；
  • 模式：传输模式（保护端到端数据）或隧道模式（保护整个数据包）；
  • 应用：VPN、站点到站点连接；
  • 安全特性：加密（ESP）、认证（AH）、密钥交换（IKEv2）。

传输层协议

SSL/TLS（Secure Sockets Layer/Transport Layer Security 安全套接字层/传输层安全协议）

  • 功能：是传输层协议，在VPN（如OpenVPN）中使用时，可间接保护链路层数据；
  • 应用：保护点对点或客户端-服务器通信；
  • 安全特性：加密、认证、完整性。

应用层协议

SSH（Secure Shell 安全外壳协议）

  • 功能：应用层安全通信协议，提供安全的远程登录、命令执行和文件传输（SFTP/SCP），支持隧道功能；
  • 模式：客户端-服务器模式：通过 TCP（默认端口 22）建立加密连接；隧道模式：支持本地、远程和动态端口转发；
  • 应用：远程服务器管理（如 Linux/Unix 服务器）、安全文件传输、VPN 隧道；
  • 安全特性：加密、认证、完整性、前向安全性。

TDS（Tabular Data Stream）

  • 功能：应用层数据库通信协议，微软和 Sybase 开发的一种专有协议，用于客户端与数据库服务器（如 Microsoft SQL Server、Sybase ASE）之间的通信；
  • 模式：客户端-服务器模式：基于 TCP/IP 的二进制协议，优化了数据库通信效率；
  • 应用：Microsoft SQL Server 的主要通信协议；
  • 安全特性：支持SSL/TLS加密；基于用户名密码认证；可集成 Kerberos 认证。

WebSocket

  • 功能：应用层通信协议，基于 TCP 的全双工通信协议，允许客户端和服务器之间建立持久连接，实时双向数据传输；
  • 模式：全双工通信：客户端和服务器可同时发送和接收数据；
  • 应用：实时应用：如在线聊天、实时游戏、股票交易平台、协作工具；推送通知、实时数据监控（如 IoT 设备）。
  • 安全特性：支持WSS（WebSocket Secure，基于 TLS/SSL）加密通信；需要验证连接来源（Origin 头）以防止跨站 WebSocket 劫持；支持认证机制（如 token 或 OAuth）。

SFTP（Secure File Transfer Protocol）

  • 功能：应用层文件传输协议，基于 SSH（安全外壳协议）的文件传输协议，用于在客户端和服务器之间安全传输文件；
  • 模式：客户端-服务器模式：通过 SSH 通道传输文件；
  • 应用：安全传输敏感文件（银行、政府、企业数据）；
  • 安全特性：基于 SSH 的加密，保障数据机密性和完整性；支持公钥认证、密码认证，防止未授权访问；提供文件校验机制（如 MD5/SHA），确保传输数据未被篡改。

数据库接口

ODBC（Open Database Connectivity）

  • 功能：数据库访问接口，一种标准化的应用程序接口（API），允许应用程序通过标准SQL语句访问不同数据库；
  • 模式：客户端-服务器模式：应用程序通过 ODBC 驱动程序与数据库服务器通信；
  • 应用：Windows系统中连接SQL Server、Oracle、MySQL等数据库；
  • 安全特性：依赖底层传输协议（如TCP/IP）实现加密；支持 Windows 集成身份验证；ODBC 3.8+ 支持 TLS。

JDBC（Java Database Connectivity）

  • 功能：数据库访问接口，Java 提供的标准 API，用于连接和操作关系型数据库；
  • 模式：客户端-服务器模式：Java 应用程序通过 JDBC 驱动程序与数据库交互；
  • 应用：Java Web应用（如Spring Boot、JSP）连接MySQL、PostgreSQL等；
  • 安全特性：支持基于用户名密码或证书认证；支持SSL/TLS加密；可配置数据库连接池（如HikariCP）防止资源耗尽。

数据安全特性

安全特性特性定义评估关注点
保密性防止数据被未授权访问分类分级、访问控制、加密、权限管理、DLP、第三方接口
完整性防止数据被篡改，保持一致性校验机制、日志审计、事务控制、异常处理、备份一致性
可用性确保数据在需要时可访问备份恢复、容灾架构、系统监控、防DDoS、高可用性设计

保密性指确保数据只被授权访问者访问，防止未授权访问、泄露或滥用。 评估工作：

  1. 数据分类分级：是否对数据进行分类（如敏感/非敏感）并按等级管理
  2. 访问控制机制：是否基于身份（RBAC/ABAC）进行权限控制
  3. 加密措施：数据在存储和传输中是否使用加密？算法是否符合标准
  4. 最小权限原则：是否对用户/系统权限按“最小可行”原则授予
  5. 第三方数据共享：是否控制第三方接口数据暴露范围及加密传输
  6. 泄露检测与响应：是否部署数据防泄露（DLP）系统，是否有响应预案

完整性指确保数据在存储、传输和处理过程中不被未授权篡改或损坏，并可验证其是否被篡改。 评估工作：

  1. 完整性校验机制：是否使用哈希（如SHA-256）或签名对关键数据做完整性验证
  2. 日志管理与审计：是否记录关键数据操作日志，并具备防篡改能力
  3. 事务一致性保障：系统在进行数据写入时是否支持事务一致性或冗余校验机制
  4. 系统异常处理机制：是否能避免断电/系统异常导致的数据损坏
  5. 备份一致性验证：是否对备份数据进行校验，确保一致性和可恢复

可用性指在需要时数据能够被授权人员访问和使用，系统能维持持续服务不中断，确保业务连续性。 评估工作：

  1. 数据备份与恢复能力：是否定期备份关键数据？能否快速恢复
  2. 高可用架构设计：是否部署负载均衡、冗余系统、故障转移机制等
  3. 容灾能力：是否具备异地灾备或云灾备，满足恢复时间目标（RTO）和恢复点目标（RPO）
  4. 抗拒绝服务能力：是否具备防DDoS、系统负载保护等机制
  5. 关键系统监控机制：是否实时监控数据访问、系统性能和可用性
